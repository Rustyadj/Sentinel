# Reflection and consolidation

Reflection runs only for significant experiences: failure/partial outcome,
three or more tools, five or more actions, human feedback, or configured
cost/runtime thresholds. Evaluator observations become candidates, never direct
memory. A skill proposal still needs independent runs and replay evidence.

Consolidation is bounded to one explicit scope and 1,000 current objects. It
identifies exact normalized-title duplicate groups and creates reviewable,
provenance-linked consolidation proposals. It does not rewrite the entire graph.

Scheduling is intentionally external. Call the authenticated consolidation and
reflection endpoints from the production scheduler. Near-duplicate embedding
clustering, automated low-value decay, and tenant-safe legacy contradiction
backfill remain incomplete.
